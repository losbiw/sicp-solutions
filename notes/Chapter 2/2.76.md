
### Explicit dispatch

Manually edit every generic procedure to account for a new type.

To account for a new procedure, create a generic procedure which again explicitly dispatches on every single type.

### Data-directed dispatch

Create a new package and expose its methods to via a get/put interface to create a new type.

Create the new method inside each of the packages and manually expose it via `put`.

### Message passing

To add a new type, create a new constructor with its own dispatch.

To add a new operation, add the handler for the operation in every types internal dispatch procedure.

## Winners

Message passing seems like the clearest way to handle both situations with most types and operations
