The issue arises because of dependency problem. The system has no way of knowing when it two unknown intervals refer to the same quantity, thus they are treated independently and their uncertainties are added up. E.g, it implicitly assumes that R1 on the top and R2 on the bottom can have different values are the same time (thus increasing overall uncertainty of the system). Same with R1/R1 division.

A package without this issue could, theoretically, be created if it could symbolically simplify algebraic expressions. E.g, for interval A: $\frac{A^6}{A^5}$ should equal just A, but with current interval arithmetic it would create a new interval with combined uncertainties of the given intervals.

However, since not all equations can be rearranged to contain only one occurrence of each variable, that wouldn't solve the problem completely either.

Potentially, if one could keep track of the identity of each interval, such a system could be implemented, but not with the current abstractions.
