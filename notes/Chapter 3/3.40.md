### No serialization

```scheme
(define x 10)  
  
(parallel-execute (lambda () (set! x (* x x)))  
                  (lambda () (set! x (* x x x))))
```

where:
- A: (* A1 A2)
- B: (lambda () (set! x A))
- C: (* C1 C2 C3)
- D: (lambda () (set! x C))
- An = nth access to x within procedure a
- Cn = nth access to x within procedure C

No serialization, thus the possible permutations are:
`((A1 A2 B C1 C2 D) (A1 A2 C1 B C2 D) (A1 A2 C1 C2 B D) (C D A B) (C A D B) (C A B D) ...)`
- A->B->C->D (1000000): A+B set x to 100, C+D set x to 100^3 (1000000)
- A->C->B->D (1000): A computes 100, C computes 1000, B sets x to 100, D sets x to 1000
- A->C->D->B (100): A computes 100, C computes 1000, D sets x to 1000, B sets x to 100
- C->D->A->B (1000000): C computes 1000, sets x to 1000, A computes 1000^2 (i.e 1000000), B sets x to 1000000
- C->A->D->B (100): C computes 1000, A computes 100, D sets x to 1000, B sets x to 100
- C->A->B->D (1000): C computes 1000, A computes 100, B sets x to 100, D sets x to 1000

Hence some possible values: 100, 1000, 1000000 

Some interleaving scenarios (too cumbersome to write out by hand):
10000: C executes in full after A1 and before A2
100000: A executes in full after C1 but before C2-3 (thus yielding 10 * 100 * 100)

### Serialized

Only options left are:
1000000: A+B compute x^2 and set x to 100, C+D compute x^3 and set x to 1000000
1000000: C+D compute x^3 and set x to 1000, A+B compute x^2 and set x to 1000000
